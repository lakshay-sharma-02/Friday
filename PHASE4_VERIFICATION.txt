RULE ENGINE BEHAVIOR
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

BLOCKING RULES (prevent execution):
  1. BLOCK_WRITES when disk >= 98%
     Logs: [rule] ✗ Disk usage critical: 98.5%
     Returns: {'blocked': True, 'message': 'Rule: Disk usage...'}

  2. TERMINATE_PROCESS when shell > 300s
     Logs: [rule] ✗ Shell command exceeded timeout: 400s
     Returns: {'blocked': True, 'message': 'Rule: Shell command...'}

WARNING RULES (log only, don't block):
  3. REDUCE_CONCURRENCY when RAM >= 95%
     Logs: [rule] ⚠ RAM usage critical: 96.2%

  4. AVOID_EXPENSIVE_WORK when battery <= 15% (not charging)
     Logs: [rule] ⚠ Battery critical: 12% (not charging)

  5. WARN_STALL when CPU >= 95% but task idle
     Logs: [rule] ⚠ High CPU usage but task appears idle

All rules respect the permission system - blocking actions create
failed log entries that surface to the user.
