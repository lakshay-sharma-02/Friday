# Friday Project Guidelines

## Documentation and Reporting

**Do not create summary, report, or documentation files** (e.g., PHASE_COMPLETE.md, EXECUTIVE_SUMMARY.md, delivery checklists, test reports) after finishing a task unless explicitly asked for documentation.

When reporting that something is done:
- Paste the actual raw terminal/test output in the response directly
- Not a bulleted summary or a checklist of claims
- If a test wasn't actually run, say so explicitly rather than marking it complete

## Testing and Verification

- Run tests and paste their raw output
- For UI/frontend changes, actually test the feature before reporting completion
- If you can't test something, state that explicitly

## Code Style

- No premature abstractions
- No features beyond what's requested
- Default to no comments (only document non-obvious WHY, not WHAT)
- Three similar lines > premature abstraction
