RULE RENAMING: COMPLETE
========================

CHANGE SUMMARY:
---------------
Renamed: TERMINATE_PROCESS → BLOCK_ON_TASK_BUDGET_EXCEEDED

REASON:
-------
The rule does NOT kill running processes. It blocks NEW steps from starting
when total task elapsed time exceeds 300s threshold.

Per-command hang killing is handled separately by tools/shell.py's timeout
mechanism (Popen + terminate()/kill()), not by the rule engine.

TWO TIMEOUT MECHANISMS (now clearly distinguished):
----------------------------------------------------

1. BLOCK_ON_TASK_BUDGET_EXCEEDED (rule engine)
   - Measures: total task elapsed time (since task started)
   - Fires: when task runs > 300s total
   - Action: blocks NEW shell commands from starting
   - Does NOT: kill already-running processes

2. Per-command timeout (tools/shell.py)
   - Measures: individual command execution time
   - Default: 30s per command (configurable)
   - Action: kills the running process (terminate → kill)
   - Proven: PID 624128 spawned, killed after 3s, gone from ps
